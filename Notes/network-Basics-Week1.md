# Networking Basics - Week 1

## Overview

Last week I began my networking journey using Cisco Packet Tracer after i was done with Introduction to Cybersecurity. My goal was to understand how devices communicate on a network and become familiar with basic networking concepts.

## Topics Covered

### Cisco Packet Tracer

* Installed Cisco Packet Tracer
* Explored the Packet Tracer interface
* Test on Pka,pkt,pksz and pkz files
* Learned the difference between the Logical and Physical workspaces
* Created simple network topologies

### IP Addressing

* Learned what an IP address is
* Understood how devices are identified on a network
* Practised basic IP configuration

### Routers

* Learned the purpose of routers
* Understood how routers help connect networks
* Explored router configuration basics

### DHCP

* Learned what DHCP stands for (Dynamic Host Configuration Protocol)
* Understood how DHCP automatically assigns IP addresses to devices
* Observed DHCP behavior in Packet Tracer

### Practical Exercises

* Built simple networks in Packet Tracer
* Connected devices together
* Tested communication between devices
* Practiced basic networking commands

## Key Lessons Learned

* Every device on a network needs an IP address.
* DHCP simplifies network management by automatically assigning addresses.
* Routers help different networks communicate.
* Packet Tracer is a powerful tool for learning networking concepts safely.

## Reflection

This was my first serious step into networking. Although some concepts were new, building and experimenting with networks in Packet Tracer helped me understand how modern networks operate. I look forward to learning subnetting, switching, network security, and more advanced configurations.
